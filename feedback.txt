## 🚨 PROBLEMAS PENDIENTES

### 1. Métricas No Se Persisten en data.json

En `data.json`, los campos `metrics` están vacíos:
```json
"metrics": {}
```

### 1. Métricas y Relevance Score

**Cómo funciona actualmente:**
- Las métricas se scrapean en **tiempo real** cuando alguien visita una página de artista (vía `/api/stats`)
- El sistema de **snapshots.json** guarda historial de métricas (funciona correctamente)
- El campo `"metrics"` en data.json siempre estuvo vacío y no se usaba

**Problema:** El Factory intenta mostrar `artist.metrics.relevance_score` pero:
- El campo `metrics` fue eliminado de data.json (no se usaba)
- El relevance_score **no se calcula ni guarda en ningún lugar actualmente**
- El Factory ahora muestra "-" porque no tiene datos

**A futuro (opciones):**
1. **Opción A:** Dejar como está - el Factory muestra "-" (no es crítico)
2. **Opción B:** Implementar persistencia de métricas en data.json
   - Cuando se scrapean métricas, guardarlas en el campo `metrics` del artista
   - Calcular y guardar `relevance_score` al mismo tiempo

**Recomendación:** Por ahora dejar como está. El sistema actual funciona bien para las páginas públicas (scraping en tiempo real). El Factory puede mostrar "-" hasta que sea necesario implementar persistencia.

### 2. Relevance Score - Código Duplicado (✅ LIMPIADO)

Existía DOS implementaciones de `calculateRelevanceScore`:
1. **rankingEngine.ts** - lógica principal
2. **componentEngine.ts** - duplicada (ahora eliminada)

**Solución:** Se eliminó la función duplicada de componentEngine.ts y ahora importa desde rankingEngine.ts. ✅

## ⚠️ PROBLEMAS MENORES

### 5. Factory Sin Autenticación (NO ES PROBLEMA)

El Factory Dashboard (`/factory`) es accesible públicamente PERO:
- En producción (Vercel): todas las operaciones de escritura (POST/PUT/DELETE) fallan porque el filesystem no está disponible
- Solo funciona completamente en desarrollo local
- Por diseño: todos los perfiles son creados manualmente por el administrador

**Estado:** No es un problema de seguridad por ahora. Si en el futuro se implementa una DB real (Supabase), ahí sí será necesario agregar autenticación.

### 6. Formulario de Aplicación - NO EXISTIRÁ

Por diseño decided:
- Los artistas NO pueden crear su propio perfil
- Todos los perfiles son creados manualmente por el administrador desde Factory
- Los interesados deben contactar por WhatsApp para pedir ser agregados

**Esto está OK así.**

### 7. Scraping Frágil

El sistema depende de:
- Scraper de Spotify (puede fallar por cambios en la UI)
- Scraper de Instagram (requiere headers especiales)
- API de TikTok (puede cambiar sin previo aviso)
- YouTube API (requiere API key en .env.local)

**Sin API key de YouTube configurada**, el sistema fallback a métodos alternativos que pueden fallar.

### 8. Producción No Persiste Cambios

En `app/api/artists/route.ts`:
```typescript
if (process.env.NODE_ENV === 'production') {
  return NextResponse.json(
    { error: 'Create artist not available in production. Use local development mode.' },
    { status: 503 }
  )
}
```

Esto significa que en Vercel (producción):
- NO se pueden crear artistas
- NO se pueden editar artistas
- NO se pueden eliminar artistas

**El proyecto solo funciona completamente en desarrollo local.**

## 🔍 ERRORES LÓGICOS ENCONTRADOS

### Error 1: Fallback Data Incorrecto
**Archivo:** `app/api/artists/route.ts:9-25`
El fallback hardcodeado muestra 25,000 listeners de Spotify pero el artista real tiene métricas diferentes.

### Error 2: Duplicación de Configuración
Hay dos fuentes de verdad para datos de artistas:
1. `data.json` - fuente principal
2. `artistConfig.ts` - archivo hardcodeado que no se usa

### Error 3: Métricas en data.json Vacías
Los campos `"metrics": {}` están vacíos porque:
- El scraping solo retorna datos a la UI
- No hay lógica para guardar en `data.json`
- Los snapshots no se implementaron completamente

### Error 4: Inconsistencia de Nombres
- En `data.json`: `"slug": "HydraRosario"` (capitalizado)
- En el código se espera slug en minúsculas para routing
- Puede causar errores 404
